# ADR 008: Рефакторинг цветовой палитры, профиля пользователя из JWT и настроек-заглушек

## Статус
Реализовано

## Контекст
В рамках улучшения интерфейса приложения и подготовки к интеграции новых фич возникла необходимость в рефакторинге:
1.  **Цветовая схема:** Убрать кастомные фиолетовые оттенки и перевести цветовую палитру приложения на стандартные системные цвета Apple (в соответствии с Apple HIG).
2.  **Профиль пользователя:** Наполнять экран профиля динамически на основе JWT-токена, полученного от бэкенда. Требовалось спроектировать единый, расширяемый формат профиля, поддерживающий разных провайдеров авторизации в будущем (Google, Apple, Email).
3.  **Локальные настройки:** Реализовать задел под интеграцию с пользовательскими настройками на бэкенде. Поскольку эндпоинты настроек уведомлений и FaceID еще не готовы, переключатели в UI должны быть временно деактивированы.

## Решение

### 1. Переход на системные цвета
Кастомный фиолетовый градиент `Color(red: 99/255, green: 102/255, blue: 241/255)` и повсеместное использование `.indigo` заменены на стандартный системный `.blue` (и сопутствующий `.cyan` в градиентах).
*   В [LoginView.swift](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/LoginView.swift) изменен фоновый ambient glow, градиент логотипа и кнопка входа.
*   В [iosApp.swift](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/iosApp.swift) изменены цвет кнопок создания домовладений, акцентные иконки в списках, вкладки `TabView` (`.tint(.blue)`) и цвет калькулятора.

### 2. Унифицированный профиль пользователя из JWT
*   В [AuthViewModel.swift](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/AuthViewModel.swift) добавлены структуры `UserProfile` и `JWTClaims`:
    ```swift
    struct UserProfile {
        let id: String
        let email: String
        let name: String?
        let avatarUrl: String?
        let provider: LoginProvider
        
        enum LoginProvider: String {
            case google = "Google"
            case apple = "Apple"
            case email = "Email"
            case unknown = "Неизвестный"
        }
    }
    ```
*   Реализована функция `decodeUserProfile(from:)` для извлечения клеймов (ID, email, name, picture) и автоматического определения провайдера авторизации по значению `iss` (issuer) или домену почты.
*   В `SettingsTab` в [iosApp.swift](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/iosApp.swift) переработан блок "Аккаунт":
    *   При наличии аватара в профиле отрисовывается изображение через `AsyncImage`, иначе отображается системная иконка-заглушка.
    *   Отображаются имя, почта пользователя и тип провайдера авторизации (например, "Google Синхронизация").

### 3. Деактивация настроек
*   В блоке "Настройки" компонента `SettingsTab` переключатели уведомлений и Face ID привязаны к константным значениям `.constant(false)` и снабжены модификатором `.disabled(true)`.
*   Добавлена визуальная плашка "Скоро" для информирования пользователя о будущей поддержке функций.

## Результаты
*   Приложение приведено в соответствие с Apple HIG благодаря использованию системного синего акцентного цвета.
*   Реализован надежный и расширяемый механизм парсинга профилей пользователей из JWT.
*   Внедрены визуальные маркеры "Скоро" для планируемых настроек FaceID и уведомлений.
