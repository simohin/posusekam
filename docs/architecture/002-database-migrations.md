# ADR 002: Управление миграциями БД (db-controller)

## Статус
Принято и реализовано

## Контекст
Необходимо декларативно управлять схемой продуктовой базы данных (PostgreSQL). Мигратор должен запускаться изолированно, не хранить пароли в коде и не держать постоянного подключения.

## Решение
Создан выделенный Gradle-модуль `db-controller` на Kotlin.
- Инструмент миграций: **Flyway**.
- Упаковка: Multi-stage Dockerfile на базе `eclipse-temurin:21-jre-alpine` (архитектура `linux/amd64` для совместимости с сервером).
- Деплой: Helm-чарт, описывающий ресурс Kubernetes `Job`. Запускается единоразово.
- Интеграция с Vault: Пароли к БД не передаются через переменные окружения K8s. Вместо этого в `Job` используются аннотации **Vault Agent Injector**, который перед стартом мигратора считывает секреты из `secret/data/posusekam/database` и формирует локальный файл `.properties` внутри контейнера. Мигратор парсит этот файл и подключается к БД.
