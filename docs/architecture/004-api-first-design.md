# ADR 004: API-First проектирование и OpenAPI генерация

## Статус
Принято и реализовано

## Контекст
С появлением микросервисов необходимо обеспечить строгую консистентность API, автоматическую генерацию DTO и контроллеров, а также иметь готовую документацию для разработки мобильных клиентов (KMP).

## Решение
Выбран **API-First (Contract-First)** подход на базе спецификации OpenAPI 3.0.
1. В каждом микросервисе спецификация хранится локально (например, `auth-service/src/main/resources/api/openapi.yaml`).
2. Создан общий модуль `core`, который содержит конфигурацию `org.openapi.generator` (`openapi-config.gradle`) и базовые зависимости (`swagger-annotations`).
3. При сборке (задача `openApiGenerate`) автоматически создаются Spring MVC интерфейсы и DTO-модели. Мы реализуем только бизнес-логику в контроллерах, имплементируя эти интерфейсы.
4. Маршрутизация настраивается через свойство `server.servlet.context-path` (например, `/auth`), чтобы каждый микросервис имел свой префикс, а Ingress Controller K3s занимался прозрачным роутингом трафика до нужного пода по префиксу.
