# ADR 011: Рефакторинг навигации в iOS-приложении и стилизация аватара профиля

## Статус
Реализовано

## Контекст
В процессе тестирования iOS-приложения на реальном устройстве были выявлены две UI/UX проблемы:
1. **Мерцание (моргание) шапки при переключении вкладок**:
   Поскольку каждая вкладка (`OverviewTab` и `CalculatorTab`) содержала свой собственный `NavigationStack` со своим `.toolbar` (селектор дома и аватар), при переключении между вкладками в `TabView` SwiftUI полностью уничтожал и пересоздавал навигационную панель. Это вызывало визуальные скачки, мерцание и сдвиги элементов шапки.
2. **Искажение и некорректный фон аватара**:
   * Системные кнопки в Navigation Bar по умолчанию накладывали серую подложку в форме горизонтального эллипса вокруг изображения аватара.
   * Ранее размеры `.frame` и обрезка `.clipShape` применялись к контейнеру `AsyncImage`, а не к самому загруженному изображению (`Image`), из-за чего внутри круга образовывались пустые отступы, а пропорции аватарки сжимались.

## Решение

### 1. Вынесение NavigationStack на уровень ContentView
*   В [`ContentView`](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/iosApp.swift#L111) весь `TabView` был обернут в единый, общий `NavigationStack`.
*   Селектор дома (Leading ToolbarItem) и кнопка аватара профиля (Trailing ToolbarItem) были перенесены в этот родительский `NavigationStack`.
*   Состояние открытия модальных окон (`activeSheet`) и диалог удаления дома вынесены в `ContentView` и связаны через `@Binding` с табами `OverviewTab` и `CalculatorTab` для открытия форм редактирования магазинов/домов прямо из дочерних экранов.
*   Из `OverviewTab` и `CalculatorTab` были полностью удалены локальные `NavigationStack` и дублирующиеся `.toolbar` элементы.
*   Для вкладки «Вычисления» сохранено динамическое управление заголовком навигации с помощью `.navigationTitle("Вычисления")`, который автоматически подхватывается внешним `NavigationStack` при выборе этой вкладки.

### 2. Сброс системных стилей кнопки и корректный рендеринг аватара
*   К кнопке аватара в тулбаре применен стиль **`.buttonStyle(.plain)`**, который полностью убирает серый системный фон-эллипс SwiftUI.
*   Логика `AsyncImage` переписана на фазовый режим (`AsyncImage(url: url) { phase in ... }`):
    ```swift
    AsyncImage(url: url) { phase in
        switch phase {
        case .success(let image):
            image
                .resizable()
                .aspectRatio(contentMode: .fill)
                .frame(width: 32, height: 32)
                .clipShape(Circle())
        // ... обработка failure / empty
        }
    }
    ```
    Это позволило наложить `.frame` и `.clipShape(Circle())` непосредственно на возвращаемый `Image`, убрав внутренние отступы и гарантируя идеальный круг.
*   Аналогичные изменения фазового рендеринга применены к аватару в самом [`ProfileView`](file:///Users/t.simokhin/IdeaProjects/posusekam/mobile/iosApp/iosApp/iosApp.swift#L970) (размером 80x80) для консистентности UI.

## Результаты
*   **Плавный UX**: Переключение вкладок происходит бесшовно. Верхняя панель (выбор дома и аватар) остается статичной и больше не мерцает.
*   **Чистый дизайн**: Аватар профиля отображается как идеальный круг без искажения пропорций и посторонних системных подложек.
*   Изменения успешно протестированы, закоммичены и отправлены в git репозиторий.
